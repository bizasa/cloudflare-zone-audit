# Output Format — Cloudflare WP Audit Report

Output theo format sau. Sử dụng Markdown, không cần HTML.

---

## Template

```markdown

**Zone:** {zone_name} | **Plan:** {plan} | **Date:** {date}

---

**30-day Stats:** {total_gb} GB total | {hit_rate}% cache hit | {total_req_m}M requests

## Hướng dẫn viết từng section

### Score emoji
- 90-100: 🟢
- 75-89: 🟡
- 60-74: 🟠
- <60: 🔴

### Viết Critical Issues
- Tối đa 5 issues trong Critical
- Mỗi issue phải có: current state, problem, fix action, và dashboard path cụ thể
- Nếu cần expression → paste chính xác, không abbreviate
- Không viết "Consider" hay "Maybe" — Critical phải dùng ngôn ngữ imperative

### Viết Action Plan
- Group theo thời gian: Hôm nay / Tuần này / Cân nhắc
- Estimate thời gian thực tế cho từng nhóm
- Checkbox format để user có thể track
- Dashboard path phải chính xác: "Security → WAF → Custom Rules → Create rule"

### Tone
- Thực tế, ngắn gọn — user là tech-savvy, không cần giải thích dài
- Số liệu cụ thể (GB, $, %) thay vì "nhiều" hay "ít"
- Honest: nếu không pull được data → nói rõ reason, không đoán
- Nếu tính tiết kiệm $ → note rõ đây là estimate

### Sections có thể bỏ nếu không có gì
- "Suggestions" → bỏ nếu không có gì để suggest
- "Critical Issues" → bỏ nếu không có critical issue (thay bằng "✅ Không có critical issues")
- "Đang tốt" → chỉ list những gì user có thể nghi ngờ, không cần list hết

### Cuối report — luôn nhắc
1. Revoke token: `dash.cloudflare.com/profile/api-tokens`
2. Ngày review tiếp theo (3 tháng)
3. Purge cache sau khi apply changes

### Thêm section sau Action Plan — Verification Commands

```markdown
## 🔬 Verification Commands
> Chạy sau khi apply changes để confirm settings hoạt động đúng

**SSL mode:**
```bash
curl -sI https://{domain} | grep -i "strict-transport\|cf-ray"
```

**Cache hit:**
```bash
curl -sI https://{domain}/wp-content/themes/style.css | grep -i "cf-cache-status"
# Expect: cf-cache-status: HIT (lần 2 trở đi)
```

**Redirect chain (phải ≤ 2 hops):**
```bash
curl -sIL http://{domain} | grep -i "http/\|location"
```

**HTTPS redirect:**
```bash
curl -sI http://{domain} | grep -i "location"
# Expect: Location: https://{domain}
```

**xmlrpc.php block:**
```bash
curl -sI https://{domain}/xmlrpc.php | grep -i "http/"
# Expect: HTTP/2 403 hoặc 301 (không phải 200)
```

**WooCommerce cart không bị cache:**
```bash
curl -sI https://{domain}/cart/ | grep -i "cf-cache-status\|cache-control"
# Expect: cf-cache-status: BYPASS hoặc DYNAMIC
```
```

### Thêm section cuối — WooCommerce Test Checklist (chỉ hiển thị nếu WP+Woo)

```markdown
## 🛒 WooCommerce Test Checklist
> Chạy sau khi apply changes. Tick từng bước trước khi đưa lên production.

- [ ] **Homepage (anonymous)** — Load, check cf-cache-status: HIT lần 2
- [ ] **Blog/product page** — Load bình thường
- [ ] **Login/logout** — Không bị redirect loop
- [ ] **Admin dashboard** — Accessible, không bị cache
- [ ] **Add to cart** — Product thêm vào cart thành công
- [ ] **Cart page** — cf-cache-status: BYPASS (không được HIT)
- [ ] **Update cart quantity** — Quantity thay đổi đúng
- [ ] **Checkout page** — Accessible, form hiển thị đúng
- [ ] **Checkout submit** — Đến được payment page (test với sandbox nếu có)
- [ ] **Payment callback** — Webhook nhận đúng (check WC logs)
- [ ] **Apply coupon** — Coupon apply thành công
- [ ] **My Account page** — Login-specific content, không bị cache chung
- [ ] **Search/filter** — Results đúng, không bị stale cache
- [ ] **Mobile view** — Responsive, không bị vỡ layout
- [ ] **Core Web Vitals smoke test** — PageSpeed Insights hoặc GTmetrix nhanh

> ⚠️ Nếu bất kỳ bước nào fail → Purge CF cache ngay và kiểm tra lại Cache Rules.
```
